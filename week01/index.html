The following are what I learned about localStorage, et SessionStorage:

JavaScript local storage uses what techies call key-value pairs. LocalStorage is a data storage type of web storage. This allows the JavaScript sites and apps to store and access the data without any expiration date. This means that the data will always be persisted and will not expire. So, data stored in the browser will be available even after closing the browser window In other word In short, all we can say is that the localStorage holds the data with no expiry date, which is available to the user even after closing the browser window..

Methods, examples, advantages and limitations of LocalStorage:

Methods:

  • setItem() This method is used to add the data through key and value to localStorage,
  • getItem() It is used to fetch or retrieve the value from the storage using the key,
  • removeItem() It removes an item from storage by using the key,
  • clear() It is used to gets clear all the storage,

Examples of LocalStorage:

  • localStorage.setItem(storyName,storyHTML), this is used to add data,
  • const res = localStorage.getItem("city"); this is used to retrieve data,
  • localStorage.removeItem("city"); this is used to remove data.
  • localStorage.clear(); this is used to clear localStorage.

Advantages of localStorage:

  • The data collected by localStorage is stored in the browser. You can store 5 MB of data in the browser.
  • You can remove all the localStorage item by a single line code, i.e., clear().
  • It also has advantages over cookies because it can store more data than cookies
  • There is no expiry date of data stored by localStorage.
  • The localStorage data persist even after closing the browser window.

Limitation of localStorage:

  • Synchronous - Each operation executes one after another.
  • Insecurity and no data protection - Can be accessed using any code.
  • Do not store sensitive information, username and password in localStorage
  • Limited storage - Can only store a maximum of 5MB data
