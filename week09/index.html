The Window Object

The window object represents a window in the browser. An object of the window is created automatically by the browser.

The window is the object of the browser, it is not the object of javascript. The javascript objects are string, array, date, etc.

Methods of window object

The important methods of window object are as follows:

MethodDescription
alert()displays the alert box containing message with ok button.
confirm()displays the confirm dialog box containing message with ok and cancel button.
prompt()displays a dialog box to get input from the user.
open()opens the new window.
close()closes the current window.
setTimeout()performs action after specified time like calling function, evaluating expressions etc.

Example of alert() in javascript

It displays alert dialog box. It has message and ok button.

            <script type="text/javascript">
            function msg(){
            alert("Hello Alert Box");
            }
            </script>
            <input type="button" value="click" onclick="msg()"/>

Output of the above example

CONFIRM - Example in javascript

It displays the confirm dialog box. It has message with ok and cancel buttons.

            <script type="text/javascript">
            function msg(){
            var v= confirm("Are u sure?");
            if(v==true){
            alert("ok");
            }
            else{
            alert("cancel");
            }

            }
            </script>

            <input type="button" value="delete record" onclick="msg()"/>

Output of the above example

Example of prompt() in javascript

It displays prompt dialog box for input. It has message and textfield.

            <script type="text/javascript">
            function msg(){
            var v= prompt("Who are you?");
            alert("I am "+v);
            }
            </script>

            <input type="button" value="click" onclick="msg()"/>

Output of the above example

Example of open() in javascript

It displays the content in a new window.

            <script type="text/javascript">
            function msg(){
            open("http://www.youtube.com");
            }
            </script>
            <input type="button" value="youtube" onclick="msg()"/>

Output of the above example

HTML5 Web Storage

The Web Storage API provides a key-value store on the client’s computer that is similar to using cookies but has fewer restrictions, more storage capacity, and is generally easier to use

HTML Web Storage

The Web Storage API provides a key-value store on the client's computer that is similar to using cookies but has fewer restrictions, more storage capacity, and is generally easier to use.

Web Storage API features crucial difference with cookies

  • Information stored is not shared with the server on every request
  • Information is available in multiple windows of the browser (if the domain is the same.)
  • Storage capacity limit is much larger tht the 4KB limit for cookies.
  • Does not automatically expire.

Websockets

Websocket is a new protocol that allows two-way communication with a server – also known as push messaging. This means that a connection is kept open and responses are ‘pushed’ to the client as soon as they are received.

Web Workers

Web workers allow processes to be run in the background, adding support for concurrency in JavaScript

To get started, use the Worker() constructor function to create a new worker:

const worker = new Worker('task.js');

Multimedia

HTML5 introduced the <audio> and <video> tags used to insert audio and video clips into a web page. It also introduced a Media API for controlling the playback of the clips using JavaScript.

An audio clip can be inserted into a page with the <audio> tag, using the src attribute to point to the audio file:

            <audio src='/song.mp3' controls>
            Your browser does not support the audio element.
            </audio>

A video clip can be inserted with the <video> tag, using the src attribute to point to the movie file:

            <video src='http://movie.mp4' controls>
            Your browser does not support the video element.
            </video>
